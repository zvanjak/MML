API & DESIGN IMPROVEMENTS
--------------------------------------------------------------------------------

  API standardization (epic jcta):
  - Standardize Config/Result/AlgorithmStatus pattern across 11 subsystems:
    derivation, integration, Fourier, linear algebra, curve fitting, ODE/DAE,
    dynamic systems, Monte Carlo, Gaussian quadrature, path/surface integration,
    and statistics
  - Add Detailed API variants with full diagnostics for all major algorithms
  - Add strong Wolfe conditions line search to BFGS optimizer

  New APIs and features:
  - Add MMLException base class for all 32 library exceptions
  - Add VectorN iterators: begin/end/cbegin/cend/data (STL-compatible)
  - Add Random::SetSeed() for reproducible simulations
  - Add std::function-based ODE system interface (ODESystemFromStdFunc)
  - Add covariant/contravariant tensor index distinction with raising/lowering
  - Add duplicate x-value validation in polynomial interpolation
  - Add static_assert for positive dimensions in MatrixNM
  - Templatize SVDecompositionSolver<Type> to match QRSolver/LUSolver
  - jacobian() now throws NotImplementedError instead of silent no-op

  Encapsulation improvements:
  - Make QRSolver fields private with public accessors
  - Make MatrixNM::_vals private
  - Make Tensor member variables private
  - Document MatrixViewNew dangling pointer lifetime risk
